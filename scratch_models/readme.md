## Layout of Config

### Option 1

<name>:
    type: <type>
    parameters:
        <param1>: <value1>

- clean and easy when there is no encapsulation

### Option 2

<name>-<type>:
    <param1>: <value1>

- flatter for encapsulated content - but harder to implement

#### Option 3

<type>:
    <param1>: <value1>

- flatter, but only viable when there is just one of each
